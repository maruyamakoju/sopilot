VigilPilot — Real-Time Violation Detection

Define Rules in Plain Text
POST /vigil/sessions { "name": "Construction Site Safety", "rules": [ "作業者がヘルメットを着用していない場合に検出", "安全ベルトなしで高所作業している人を検出", "立入禁止エリアへの侵入を検出" ], "severity_threshold": "warning", "backend": "claude" }
No ML training. Rules are evaluated frame-by-frame by the VLM. Change rules in seconds without redeployment.
Violation Event Response
// GET /vigil/sessions/{id}/events { "event_id": 42, "timestamp_sec": 183.5, "violations": [{ "rule": "ヘルメット未着用", "severity": "critical", "confidence": 0.94, "bboxes": [[234, 112, 498, 287]] }], "frame_url": "/vigil/events/42/frame?annotate=true" }

Backend Selection Guide
⚡
Fastest to deploy
Use claude — API key only, zero local infrastructure
🚫
Air-gapped / fully offline
Use qwen3 local — no external calls after model download
⚖
Cost optimization
Use qwen3-api via Together.ai / vLLM cluster
